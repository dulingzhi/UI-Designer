## 需求与现状 (Status Quo)
大多数魔兽 RPG 作者目前都是在 VSCode 里手写 `.fdf` 或者基于 API 写 UI 逻辑，编译后通过本地开房间进游戏看效果。位置差了 0.001？对齐失败了？切出来改代码，重新打包，重新进游戏。这个循环长达数分钟，完全依赖经验盲猜坐标。这就是妥妥的 “status quo”。只要这个恶心的开发闭环依然是 2005 年的水平，你的编辑器就是降维打击，它将吃掉所有写 UI 的作者的时间碎片。

## 目标与最小切入点 (Target User & Narrowest Wedge)
**The Target**: 想要高频发布、实现酷炫 UI（背包、技能树、弹窗商店），并且被长反馈链彻底逼疯的魔兽 RPG 作者。
**The Wedge**: “所见即所得 + 热重载” (A + B)。在编辑器里长得什么样，进了游戏必须 100% 长什么样；我在编辑器里拖拽一个按钮，游戏里的按钮能**立刻跟着动**。作者愿意为了这两个爽点，立刻放弃手写代码的习惯。

## 核心前提 (Premises)
1. 把 FDF 的文本抽象结构，在这个 Web/Tauri 里面 100% 完美克隆出来。即使有细微瑕疵，也会失去信任。我们需要连九宫格拉伸方式、文字截断渲染都在前端引擎里分毫不差地呈现。
2. 魔兽 RPG 引擎支持通过 Lua/Jass 实时替换/隐藏 UI （KKWE 或重制版机制），因此你的热重载代码生成器并不是梦，而是可以通过底层 WebSocket / 魔兽 I/O 文件热读来实现毫秒级刷新。
3. 即使最后我们要搞生态，必须先通过硬核的图形一致性和全栈联动拿住核心作者。

## 长期迭代路线 (Recommended Approach: A + B 并行)

基于你的选择，以下是未来 3 年必须跨越的四大核心 Milestone。这不是随便列的功能，而是每一个做出来，就能让你的留存率翻倍的增长节点。

### Milestone 1: 渲染引擎 100% 还原本源 (The WYSIWYG Purist)
这部分是最硬的骨头，Web 端的 div 和 CSS 必须退居幕后，真正的渲染应该是仿魔兽引擎。
- **BLP 与 MDX 直渲 WebGL 化**：利用 Rust 解析器读取魔兽原生材质并在前端用 WebGL 渲染，彻底抛弃传统的 PNG 转换或者 CSS Mock。你在 `RUST_MDX_BLP_IMPLEMENTATION.md` 里有开始，做到底。
- **原生 FDF 解析级一致**：Backdrop（九宫格边框像素）、Frame（锚点对齐与偏移逻辑），前端布局系统不能用 Flex 或 Grid 模拟了，必须重写一套符合魔兽内部 Absolute/Relative/SetPoint 定位规则的布局系统核心。
- **字体渲染同步**：把魔兽字库抽出来，在 Canvas 层面实现真正的字距、行距模拟，做到“多一个字截断，在编辑器里必然也截断”。

### Milestone 2: 毫秒级热重载长连接 (The Hot-Reload Kill Shot)
光有 Lua 生成和 `-reload` 机制不够。我们要干掉敲命令这一步。
- **Tauri WebSocket 同步侧**：利用 Tauri 的本地服务能力，不仅监控项目文件，而且可以直接创建一个 `locahost` 服。
- **Lua 热跟端组件**：提供给作者的一个“开发期特供” Lua 脚本底座，游戏内实时通过 Http 请求获取到你的设计器最新下发的坐标 Diff。
- **效果**：作者在这个软件里鼠标长按拖走一个按钮，旁边屏幕的魔兽争霸游戏里，按钮以 60fps 跟着移动。这个一旦做成视频发出去，$100% 的魔兽圈开发者都会下载你的软件。

### Milestone 3: 完整的双向解析能力与组件化 (The Semantic AST)
目前的导出是不可逆的单向生成，未来要成为主流工具，意味着必须兼容历史包袱。
- **FDF 的无损反向解析**：用户拖进一个暴雪原生 20 年前写的 `ConsoleUI.fdf`，你必须能 100% 把它渲染到 Web 界面，且节点层级（Frame 父子继承）全部可视。
- **组件/预制体生态（Prefab）**：在这个基础之上，加入现代 UI 最好的部分——像 React 组件一样，打包某个复杂的技能树为 “Prefab”，方便多个地图复用分享。

### Milestone 4: 开发生态集成 (The Ecosystem Extender)
它现在是个独立程序，未来它应该是顺应流的工具链。
- **VSCode Webview 扩展嵌入**：将 Tauri 的网页端抽取出来写成一张大号版的 VSCode Extension，因为大家都在 VSCode 写代码。让编辑 UI 只是其中一个 Tab 的事。
- **版本控制与 Diff 预览**：可视化支持基于 `.w3ui` 格式的 Git Diff 显示（比如图例显示按钮坐标发生了什么变动）。

## The Assignment (你的下一步破局任务)

不要去铺更多的基础控件类型，或者做界面美化。
**证明 Milestone 2 可以做成 60帧 的幻觉。**
去建立一条极简的本地通道，在 Web 端写一个滑块，在魔兽里的一个方块就跟着滑块实时的 X 轴移动。如果你能录制这样一段 10 秒的视频，你的这个切入点（Wedge）就证成了。你不需要一整套 UI 设计工具，只需要一个滑块和一个实时移动的按钮，就能证明技术可行性。做完这个后，再反哺所有的控件。

## What I noticed about how you think
- **你知道什么是真正的痛点**：你没说“我要帮他们画好看的 UI”，你直指“编译、进游戏、改代码”这个折返跑循环。这是极其典型的创始人嗅觉。
- **你敢于触碰魔兽底层机制**：通过外部软件控制 KKWE 热重载是不容易的，但你选择这套 A+B 方案，说明你不惧怕硬工程（Hard-tech），这不是随便套个网页壳子就拿去骗人的。
